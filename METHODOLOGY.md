# Отчет: Алгоритм поиска геометрической медианы

## 1. Постановка задачи
Нужно найти такую точку $P(x, y)$ на плоскости, для которой суммарное евклидово расстояние до всех заданных точек $P_i(x_i, y_i)$ будет минимальным.

## 2. Математическое обоснование
Это классическая задача поиска **геометрической медианы** (также известной как точка Ферма-Вебера).

Важно не путать ее с центроидом. Центроид минимизирует сумму квадратов расстояний, а геометрическая медиана — сумму самих расстояний. Из-за этого закрытой формулы для общего случая нет, поэтому решение обычно ищется численно.

Целевая функция:
$$f(x, y) = \sum_{i=1}^{n} \sqrt{(x - x_i)^2 + (y - y_i)^2}$$

## 3. Выбор алгоритма и System-ready подход
В задачах Data Science подобные проблемы часто закрывают готовыми оптимизаторами из `scipy.optimize`. Для системной разработки это не лучший путь: вместе с удобной оберткой приходят тяжелые бинарные зависимости (BLAS/LAPACK/Fortran), которые усложняют перенос и сопровождение на целевых платформах.

По этой причине выбран **алгоритм Вайсфельда (Weiszfeld's algorithm)** и реализован вручную. Это итеративный метод, в котором новая оценка точки вычисляется как взвешенное среднее, где веса обратно пропорциональны расстояниям до исходных точек. Такой подход прозрачен, предсказуем и хорошо переносится на C/C++.

## 4. Реализация и работа с памятью
* В качестве начального приближения используется центроид.
* Вычислительное ядро (`geometric_median_math.py`) написано в процедурном стиле (pure functions) и работает с типизированными contiguous-массивами `numpy` формата `float64`.
* Отдельно обработан случай сингулярности (деление на ноль): если текущая оценка совпадает с одной из входных точек, алгоритм корректно завершает работу.
